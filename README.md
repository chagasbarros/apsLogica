# Digital Security Academy 🛡️

Uma aplicação Java de console projetada para promover a **Inclusão Digital**, focada em ensinar práticas de segurança cibernética para usuários. Este projeto faz parte dos estudos de **Lógica de Programação**.

##  Sobre o Projeto

O **Digital Security Academy** funciona como um guia interativo onde especialistas (instrutores) ensinam conceitos fundamentais de proteção online. O objetivo principal é transformar termos técnicos complexos em passos simples e compreensíveis, reduzindo a vulnerabilidade de novos usuários no ambiente digital.

## 🚀 Funcionalidades Atuais

*   **Validação de Senhas Fortes:** Um guia passo a passo que verifica em tempo real:
    *   Comprimento mínimo (8 caracteres).
    *   Uso de letras maiúsculas e minúsculas.
    *   Presença de números.
    *   Uso de caracteres especiais.
*   **Menu Interativo:** Navegação simplificada via console.
*   **Feedback Imediato:** Mensagens claras para orientar o usuário em caso de erro, seguindo princípios de usabilidade.

## 🛠️ Tecnologias Utilizadas

*   **Linguagem:** Java 17+
*   **Entrada de Dados:** `java.util.Scanner`
*   **Estruturas de Controle:** `Switch Case` moderno (Java 14+) e loops `while` e tomada de decisão `if`.
*   **Validação:** Expressões Regulares (Regex).

## 📂 Estrutura do Código

Seguindo as boas práticas de programação:
*   **Modularização:** O código é dividido em métodos para facilitar a manutenção e os testes.
*   **Nomenclatura:** Segue o padrão *camelCase* para métodos/variáveis e *PascalCase* para classes.

## 🔧 Como Executar

1. Certifique-se de ter o **JDK** instalado em sua máquina.
2. Clone o repositório:
   ```bash
   git clone https://github.com/seu-usuario/digital-security.git
   ```
3. Compile o arquivo:
   ```bash
   javac DigitalSecurity.java
   ```
4. Execute a aplicação:
   ```bash
   java DigitalSecurity
   ```

## 🧠 Conceitos de IHC Aplicados

Este projeto aplica as Heurísticas de Nielsen, como:
1.  **Prevenção de Erros:** O sistema valida a senha antes de permitir que o usuário avance.
2.  **Ajuda e Documentação:** Instruções claras sobre o que compõe uma senha segura.
3.  **Feedback:** Mensagens de sucesso e erro que educam o usuário em vez de apenas puni-lo.
